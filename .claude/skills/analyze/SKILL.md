# 企業決算分析スキル



- ファイル名は `決算年度-決算月-区分.pdf` 形式（例: `2025-3-q4.pdf`）
  - 決算年度: 西暦4桁
  - 決算月: 月（ゼロ埋めなし）
  - 区分: q1/q2/q3/q4（四半期）
- ディレクトリ内のPDFをリストアップし、**ファイル名の降順でソート**する
- **期間の決定ルール**:
  - ユーザーが期間を指定した場合（例: 「1年分」「直近2年」「2023年以降」）: その期間に合うPDFを選ぶ
  - 指定がない場合: **直近3年分**（最新ファイルの決算年度から3年前以降）を選ぶ
  - ファイルが少ない場合はある分だけ使う
- 対象ファイルが12件を超える場合は、ユーザーに確認してから進む

## Step 2: 並列PDF読み込み（複数subagent）

対象PDFそれぞれについて、**同時に**subagentを起動して内容を読み込む。

**重要**: PDFの読み込みには必ず `Read` ツールを使用すること。

各subagentへの指示：

```
以下のPDFファイルをReadツールで読み込み、決算短信の内容を構造化して返してください。
外部コマンド（pdftotext等）は使わず、必ずReadツールを使ってください。

ファイル: company/<証券コード>/<決算年度-決算月-区分>.pdf

抽出すべき情報:
- 決算期間（何年何月〜何月期）
- 売上高（前年同時期と比較）
- 営業利益（前年同時期と比較）
- 純利益（前年同時期と比較）
- セグメント別売上（あれば）
- 主要な事業トピック・注記
- 業績予想の修正（あれば）
```

## Step 3: 分析subagentの並列起動

Step 2の結果が揃ったら、以下の2体のsubagentを**同時に**起動する。

---

### Subagent A: 事業解説（わかりやすい説明）

```
あなたは高校生にもわかりやすく企業を説明するアナリストです。
以下の決算データをもとに、企業の事業内容と業績トレンドを解説してください。

[Step 2の読み込み結果を渡す]

出力形式（Markdown）:

## この会社は何をやっている会社？
（200字程度で、中学生でもわかる言葉で説明）

## どうやってお金を稼いでいる？
（売上の内訳・セグメントを図や箇条書きで）

## 最近の業績トレンド
（直近5四半期の売上・利益の推移をテーブルで。増えてる？減ってる？）

## 注目ポイント
（今後の見通し・気になる点を3つ以内で箇条書き）
```

---

### Subagent B: ファンダメンタル投資分析

```
あなたは長期投資家の視点で企業を分析するアナリストです。
以下の決算データと投資チェックリストをもとに、投資価値を分析してください。

[Step 2の読み込み結果を渡す]
[investment_checklist.md の内容を渡す]

出力形式（Markdown）:

## 投資チェックリスト判定

| 項目 | 結果 | 根拠（数値） |
|------|------|-------------|
| 売上高成長率（YoY）≥5% | ✅/❌ | XX% |
| 利益成長（YoY） | ✅/❌ | XX% |

## 四半期別パフォーマンス
（最新4四半期 + 前年同時期の5件を縦に並べたテーブル）

## 投資判断サマリー
（チェックリストの合否をもとに「買い候補」「要観察」「見送り」の3段階で判定し、その理由を200字程度で）

## リスク要因
（財務データから読み取れる懸念点を3つ以内で箇条書き）
```

## Step 4: レポートの保存

両subagentの出力を統合し、以下のパスに保存する。

```
company/<証券コード>/analysis/YYYYMMDD.md
```

（`YYYYMMDD` は実行した日付。例: `20250411.md`）

### レポートの構成

```markdown
# <企業名>（<証券コード>）決算分析レポート
分析日: YYYY年MM月DD日
対象期間: <最古のPDF期間> 〜 <最新のPDF期間>
使用データ: <使用したPDFファイル名（例: 2025-3-q4.pdf）をカンマ区切りで>

---

## Part 1: この会社をわかりやすく理解する

（Subagent Aの出力をそのまま）

---

## Part 2: 投資家目線のファンダメンタル分析

（Subagent Bの出力をそのまま）
```

---

## エラーハンドリング

- 対象PDFが少ない場合: ある分だけ分析し、レポートに「データ数: N件」と明記
- PDFが読み込めない場合: そのファイルをスキップしてレポートに記載
- `company/<証券コード>/` が存在しない場合: ユーザーに証券コードを確認
